# Module 2: Digital Twin - Weekly Progression

## Module Overview
Module 2 covers simulation environments for robotics, focusing on Gazebo Garden and Unity as digital twin platforms. This module builds on ROS 2 fundamentals from Module 1 and prepares students for advanced simulation with Isaac in Module 3.

## Week 1: Introduction to Gazebo Simulation

### Topics Covered
- Gazebo Garden installation and setup
- Basic Gazebo concepts: worlds, models, plugins
- Integration with ROS 2 using gazebo_ros_pkgs
- Creating simple simulation environments
- Basic robot spawning in simulation

### Learning Objectives
- Install and configure Gazebo Garden
- Understand Gazebo's architecture and components
- Create basic simulation worlds
- Integrate ROS 2 nodes with Gazebo simulation
- Spawn and control simple robots in simulation

### Prerequisites
- Completed Module 1 (ROS 2 fundamentals)
- Understanding of URDF robot modeling
- Basic ROS 2 communication patterns

### Activities
- Install Gazebo Garden and verify setup
- Run basic Gazebo tutorials with ROS 2
- Create a simple world file with static objects
- Spawn a pre-built robot model in simulation

### Assessment
- Successfully run a basic Gazebo simulation with ROS 2 integration
- Demonstrate understanding of Gazebo components through Q&A
- Create and run a simple custom world file

## Week 2: Advanced Gazebo Concepts and Physics

### Topics Covered
- Physics engines in Gazebo (DART, Bullet, ODE)
- Creating custom robot models for simulation
- Sensor simulation (cameras, LIDAR, IMU)
- Physics properties and material definitions
- Collision and visual properties

### Learning Objectives
- Configure physics properties for realistic simulation
- Create custom robot models suitable for simulation
- Implement sensor simulation in robots
- Understand the impact of physics settings on simulation
- Tune physics parameters for specific applications

### Prerequisites
- Week 1 content mastery
- Understanding of URDF/SDF conversion
- Basic knowledge of robot sensors

### Activities
- Create a custom robot model with sensors
- Configure physics properties for the robot
- Implement sensor plugins for the robot
- Test robot behavior with different physics settings

### Assessment
- Successfully simulate a custom robot with sensors
- Demonstrate proper physics configuration
- Show sensor data integration with ROS 2

## Week 3: Unity for Robotics and Digital Twins

### Topics Covered
- Unity installation and Robotics setup
- Unity Robotics Hub tools and packages
- ROS integration via TCP connector
- Basic Unity scene creation for robotics
- Sensor simulation in Unity

### Learning Objectives
- Set up Unity for robotics applications
- Understand Unity's role in digital twin applications
- Integrate Unity with ROS 2 systems
- Create basic robotics scenes in Unity
- Implement sensor simulation in Unity environment

### Prerequisites
- Weeks 1-2 content mastery
- Understanding of ROS 2 communication patterns
- Basic concepts of 3D environments

### Activities
- Install Unity Hub and Robotics packages
- Set up ROS-TCP-Connector
- Create a simple robotics scene in Unity
- Integrate Unity scene with ROS 2 nodes

### Assessment
- Successfully connect Unity to ROS 2
- Demonstrate basic robot control in Unity
- Show sensor data flow from Unity to ROS 2

## Prerequisites for Module 3
- Strong understanding of simulation concepts
- Experience with both Gazebo and Unity
- Ability to create and configure simulation environments
- Understanding of sensor simulation and integration

## Integration with Other Modules

### Connection to Module 1 (ROS 2)
- Deepens understanding of ROS 2 communication in distributed systems
- Provides practical application of ROS 2 tools (RViz, rqt) with simulation
- Reinforces node architecture with simulation nodes

### Connection to Module 3 (Isaac)
- Foundation for more advanced Isaac Sim concepts
- Understanding of physics simulation applies to Isaac Sim
- Sensor simulation concepts transfer to Isaac ROS packages

### Connection to Module 4 (VLA)
- Simulation provides safe environment for testing voice-controlled robots
- Sensor simulation supports VLA perception systems
- Robot behavior validation in simulation for VLA commands
